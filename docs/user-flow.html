This page shows the current Divi5Generate workflow as defined by the skills: start from a brand source or brief, compile a page spec, validate the output, preview in real Divi, then publish and verify the live WordPress page.

Preferred build path page-spec.json compiles to preview HTML and Divi JSON.
Blocking quality gates Spec, validation, taste, fidelity, preview, and live visual checks.
Import behavior The importer previews in real Divi, then imports with status: publish.
No repo output Generated artifacts belong in DIVI5_OUT or ~/Desktop/Divi5 Pages.

Install First

There are two installs: the Claude skills plugin on your machine, and the WordPress site connector on each target Divi site.

Claude skills plugin

Install this once so Claude Code can load the Divi5Generate skills and commands.

claude plugin marketplace add Kieransaunders/Divi5Generate claude plugin install divi5generate@divi5generate
  1. Open or restart Claude Code Desktop after installation.
  2. Run /reload-plugins after plugin updates.
  3. Use /divi5generate:launch to open the local generator app.

WordPress site connector

Install the Divi Tools Connector on each target WordPress site. It previews and imports pages, moves presets and global variables, exports site design data, writes SEO/schema, and supports managed page cleanup.

/divi5generate:help
  1. Build or download divi-tools-connector.zip.
  2. In WordPress Admin, go to Plugins -> Add New -> Upload Plugin.
  3. Activate Divi Tools Connector.
  4. Open Settings -> Divi Tools Connector and copy the Site URL and API key.
  5. Give those to import-to-local, brand-extract, or brand-deploy when Claude needs to work with that site.

Choose the Input Path

Users can start from a live site, an export, a brand guide, a Claude Design hand-off, or a fresh brief.

Existing Divi export

  1. Run divi5-extract-style.
  2. Reuse tokens, variables, presets, and designer IDs.
  3. Generate the new page against that design system.

Live WordPress site

  1. Run brand-extract with site URL and API key.
  2. Save a Brand Profile in the local app.
  3. Reuse or deploy it with brand-deploy.

Brand guide or fresh brief

  1. Convert explicit tokens with divi5-extract-style.
  2. Or go straight to divi5-page-generator.
  3. Use the app when the user wants a chat-first workflow.

End-to-End Flow

The lanes show the current skill-defined sequence. Yellow cards are blocking checks; green cards are publish or delivery points.

Build Gate Publish Fix loop
Brief

Resolve source

Identify whether the user has an export, live site, brand guide, Claude Design hand-off, or only a brief.

Gather essentials

Brand, offer, primary keyword, location, sections, CTA, and aesthetic direction.

Spec

Author page-spec.json

Use the spec as the editable source for sections, copy, presets, SEO, and motion.

Validate spec

validate-spec.js blocks unsupported vocabulary before Divi JSON exists.

Preview

Compile HTML

spec-to-html.js produces the approval mockup from the same spec.

Approve or refine

Interactive runs wait for approval. Headless brief mode can self-approve.

Generate

Compile Divi output

spec-to-divi.js writes layout JSON, SEO sidecar, and schema files.

Run build gates

validate.js, taste-check.js, and fidelity-check.js must pass.

Import

Preview in real Divi

import-to-local posts to /preview so Divi renders presets and CSS before import.

Publish and verify

Full import uses status: publish, then screenshots and visual checks close the loop.

Quality Gates

These checks explain why a user may be sent back to refine instead of receiving a final file or URL.

Spec compatibility

Required

validate-spec.js checks the spec vocabulary before HTML or JSON compilation.

Blocks on unsupported page-spec fields.

Structural and SEO validation

Required

validate.js checks Divi structure, SEO sidecar, headings, and page rules.

Blocks on every FAIL.

Taste and fidelity

Required

taste-check.js and fidelity-check.js protect visual quality and mockup match.

Blocks on design quality or preview mismatch.

Style consistency

Conditional

divi5-style-check runs when a designer export is available.

Blocks on preset drift or off-palette output.

Real Divi preview

Required

/preview catches render issues before creating or updating the final page.

Blocks when the user rejects the render.

Live visual check

Required

Playwright screenshots and visual diff compare the live page against the approved mockup.

Blocks on visible render drift.

Spec compliance

Conditional

design-review --spec checks the exported live page against the brief.

Blocks on missing sections, CTAs, or required copy.

Output discipline

Required

Generated pages, previews, SEO, schema, and scripts stay outside the repo.

Blocks on writing artifacts to project source.

What Users Should Remember

Short operational rules for support docs, onboarding, and hand-off conversations.

Start with the best sourceLive site, export, brand guide, design hand-off, or fresh brief.
Preview before importThe approved HTML mockup is not the same as real Divi render.
Publish is intentionalImporter uses live pages because Divi draft previews are not reliable enough.
Fix from the sourceRefinements go back to the spec or generator inputs, not hand-edited JSON.
